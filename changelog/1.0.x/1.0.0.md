# 1.0.0 — 2026-06-12

Complete rebuild replacing the previous 0.3.x tool surface. The old `pixoo-text`, `pixoo-compose`, `pixoo-control`, and `pixoo-push-image` tools are removed; all new tools use the `pixoo_` prefix.

## Added

- **`pixoo_display_text`** — render styled text (theme, gradient, shadow, outline, auto-fit) onto the display and push it; returns the rendered frame as an image content block.
- **`pixoo_compose_scene`** — full layered scene composition: `text`, `icon`, `rect`, `circle`, `line`, `progress`, `sparkline`, `bitmap`, `pixels`, `image`, `sprite` elements; named effect presets or raw keyframe arrays; static PNG or animated GIF output.
- **`pixoo_push_image`** — load an image (absolute local path or `https://` URL) with URL fetch sandboxing and output-path validation, resize to the LED grid, and push.
- **`pixoo_overlay_text`** — set or clear a device-native scrolling text overlay; overlays persist across channel switches until cleared.
- **`pixoo_control_device`** — read or change device state (brightness, screen on/off, channel, clock face); call with no params for a status read.
- **`pixoo_discover_devices`** — find Pixoo devices on the local network via Divoom's cloud discovery endpoint.
- **`pixoo_design_brief`** — return craft guidance and live device context for a design topic: legibility rules, palette discipline, layout zones, animation budget, pre-filled next-tool suggestions.
- **`pixoo://device/status`** resource — live snapshot of the connected display.
- **`pixoo://themes`** resource — named theme catalog with palette data.
- **`pixoo://icons`** resource — built-in icon registry by category.
- **`pixoo://design-guide`** resource — full rendering craft reference.
- **Pure renderer pipeline** (`src/renderer/`) — RGBA canvas, styled-text engine (gradient ramps, shadow, outline, semantic alignment), theme/palette registry, icon registry, effect compiler (presets → keyframes), keyframe interpolation, preview encoder (PNG / contact-sheet / GIF); independently unit-testable without device hardware.
- **`PixooService`** (`src/services/pixoo/`) — wraps `@cyanheads/pixoo-toolkit`; handles lazy init, command pacing (`PIXOO_PUSH_MIN_INTERVAL_MS`), `ensureCustomChannel()`, result→error-contract mapping, and device snapshots.
- 166 tests covering renderer units, tool handlers, resource handlers, and service logic.
- `manifest.json` for one-click Claude Desktop install via `.mcpb` bundle.
- `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json` for marketplace listings.

## Removed

- **`pixoo-text`** tool — replaced by `pixoo_display_text`.
- **`pixoo-compose`** tool — replaced by `pixoo_compose_scene`.
- **`pixoo-control`** tool — replaced by `pixoo_control_device`.
- **`pixoo-push-image`** tool — replaced by `pixoo_push_image` (same function, stricter security).
- All previous transport, container, storage, graph, LLM, and speech scaffolding from the 0.x scaffold — superseded by `@cyanheads/mcp-ts-core` framework internals.

## Migration

Consumers of `0.3.x` must update all tool calls to use the new `pixoo_` prefix names. All parameters were redesigned; refer to the tool descriptions in the README or via MCP tool listing.
